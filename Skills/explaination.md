# Skill: Explaining Concepts

When asked to explain a concept or technology, respond using the following structure in plain, everyday language.

## Structure

### 1. Background (why it exists)

Start with what the world looked like before this concept existed — what developers were doing, what the common setup was. Do not open with a definition. Open with context.

### 2. The Problem

Describe the specific pain point the old approach created. Be concrete. Use a short scenario or example to give the reader something to relate to.

### 3. How It Solves the Problem

Explain the core idea behind the concept or technology — how it addresses the problem described above. Focus on the central insight, not an exhaustive list of features.

### 4. One-Line Summary (optional)

End with the simplest possible definition. Place it last, not first — by this point the reader has enough context for it to actually land.

## Writing Principles

- Lead with analogies and everyday examples; introduce jargon only after the reader understands the idea
- One idea per paragraph
- Aim for understanding, not completeness — leave edge cases and advanced details for follow-up
- If the concept has layers, cover the core use case first
